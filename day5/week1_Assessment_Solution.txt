Que 1 
 List stores ordered, changeable items and allows duplicates. Example: use a list for a to-do app where tasks can be added, removed, or reordered. 
Tuple stores ordered but unchangeable items. Example: use a tuple for fixed coordinates like (x, y) because they should not change accidentally. 
Set stores unique items only and is unordered. Example: use a set to remove duplicate email IDs from a collection quickly. 
Dict (dictionary) stores data in key: value pairs for fast lookup. Example: use a dict for student records like {"name": "Rahul", "marks": 90} where each key has a related value. 
Choose a list when order and editing matter, a tuple when data should stay fixed, a set when uniqueness matters, and a dict when you need labeled data with quick access.

import json
def read_json_safely(path):
    try:
        with open(path, "r") as file:
            return json.load(file)
    except FileNotFoundError:
        return {}
    except json.JSONDecodeError:
        return None

import requests
import sys
url = "https://api.github.com/users/anthropics"
try:
    response = requests.get(url)

    if response.status_code != 200:
        print(f"Error: Request failed with status code {response.status_code}")
        sys.exit()

    data = response.json()

    print("Name:", data["name"])
    print("Public Repos:", data["public_repos"])
    print("Followers:", data["followers"])

except requests.RequestException as e:
    print(f"Request error: {e}")
    sys.exit()

API keys should never be hard-coded because they can accidentally get pushed to GitHub, leaked to others, or stolen from shared code. 
The recommended approach is to store secrets in a .env file and load them using the python-dotenv package. 
You call the load_dotenv() function to load variables from the .env file into the environment.

from dotenv import load_dotenv
import os

load_dotenv()

api_key = os.getenv("OPENAI_API_KEY")

from pydantic import BaseModel, Field
from typing import List


class WeatherReport(BaseModel):
    city: str = Field(..., min_length=1)
    temperature_c: float = Field(..., ge=-50, le=60)
    humidity: int = Field(..., ge=0, le=100)
    conditions: List[str]


# Valid example
report = WeatherReport(
    city="Jaipur",
    temperature_c=31.4,
    humidity=42,
    conditions=["sunny", "windy"]
)

print(report)


# Invalid example (will fail validation)
bad_report = WeatherReport(
    city="",
    temperature_c=75,
    humidity=120,
    conditions=["hot"]
)

Que 13

Props are data passed from a parent component to a child component, and they are read-only. 
State is data managed inside a component that can change over time. 
Use props when sending information like a username or profile image to another component. 
Use state when handling dynamic data like a counter, form input, or light on/off toggle. 
Example of props: <Profile name="Sudhanshu" /> 
Example of state: const [count, setCount] = useState(0) for a button click counter.

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  const increase = () => {
    setCount(count + 1);
  };

  const decrease = () => {
    if (count > 0) {
      setCount(count - 1);
    }
  };

  const reset = () => {
    setCount(0);
  };

  return (
    <div>
      <h1>{count}</h1>

      <button onClick={increase}>+1</button>

      <button onClick={decrease}>-1</button>

      <button onClick={reset}>Reset</button>
    </div>
  );
}

export default Counter;

function UserList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://api.example.com/users")
      .then((r) => r.json())
      .then((data) => setUsers(data));
  }, []);

  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>{u.name}</li>
      ))}
    </ul>
  );
}

export default UserList;

import { useEffect, useState } from "react";

function SearchableList() {
  const [users, setUsers] = useState([]);
  const [search, setSearch] = useState("");
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState("");

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then((response) => {
        if (!response.ok) {
          throw new Error("Failed to fetch users");
        }

        return response.json();
      })
      .then((data) => {
        setUsers(data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  const filteredUsers = users.filter((user) =>
    user.name.toLowerCase().includes(search.toLowerCase())
  );

  if (loading) {
    return <p>Loading...</p>;
  }

  if (error) {
    return <p>Error: {error}</p>;
  }

  return (
    <div>
      <input
        type="text"
        placeholder="Search users..."
        value={search}
        onChange={(e) => setSearch(e.target.value)}
      />

      {filteredUsers.length === 0 ? (
        <p>No users found</p>
      ) : (
        <ul>
          {filteredUsers.map((user) => (
            <li key={user.id}>{user.name}</li>
          ))}
        </ul>
      )}
    </div>
  );
}

export default SearchableList;
